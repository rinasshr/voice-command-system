# Система распознавания голосовых команд

## 1. Схема решения

Система построена по клиент-серверной архитектуре:

```
┌─────────────┐     HTTP/REST      ┌──────────────┐     SQLite      ┌────────────┐
│   Frontend  │ ◄──────────────►   │   Backend    │ ◄─────────────► │     БД     │
│  (React)    │                    │  (FastAPI)   │                 │  (SQLite)  │
│  Vite SPA   │                    │              │                 └────────────┘
└─────────────┘                    │  ┌────────┐  │
       │                           │  │  VOSK  │  │     ┌────────────┐
       │  MediaRecorder API        │  │  ASR   │  │────►│   Файлы    │
       │  (запись с микрофона)      │  └────────┘  │     │  (uploads) │
       └──────────────────────────►│              │     └────────────┘
              WebM/Opus            └──────────────┘
```

### Поток данных:
1. Оператор авторизуется через форму логина (JWT-токен)
2. На главной странице нажимает «Начать запись» — браузер записывает аудио через MediaRecorder API
3. По нажатию «Остановить» аудиофайл (WebM) отправляется на сервер
4. Сервер конвертирует WebM → WAV (ffmpeg), распознаёт речь (VOSK), извлекает команду и идентификатор
5. Результат + аудиофайл сохраняются в БД
6. Результат отображается пользователю
7. Оператор может прослушать запись, скорректировать результат или подтвердить его

## 2. Технологии

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Backend | Python 3.11 + FastAPI | Быстрый async-фреймворк, встроенная OpenAPI-документация |
| ASR | VOSK (vosk-model-small-ru-0.22) | Offline-распознавание, поддержка русского, open-source |
| БД | SQLite + SQLAlchemy ORM | Лёгкая встраиваемая БД, достаточна для MVP |
| Аутентификация | JWT (python-jose) + bcrypt | Стандартный подход для SPA, безопасное хеширование |
| Frontend | React 18 + Vite | Быстрая сборка, компонентный подход |
| Аудио | Web MediaRecorder API | Нативная запись в браузере, без дополнительных библиотек |
| Конвертация | ffmpeg | Конвертация WebM → WAV 16kHz для VOSK |
| Деплой | Docker + docker-compose | Воспроизводимое развёртывание |

## 3. Структура базы данных

### 3.1 Инфологическая модель (ER-диаграмма)

```
┌──────────────────────────┐         ┌──────────────────────────────────┐
│         User             │         │         VoiceRecord              │
├──────────────────────────┤         ├──────────────────────────────────┤
│ PK id: INTEGER           │ 1    * │ PK id: INTEGER                   │
│    username: VARCHAR(100) │◄──────┤ FK user_id: INTEGER              │
│    hashed_password: TEXT  │        │    audio_path: VARCHAR(500)      │
│    role: VARCHAR(20)      │        │    raw_text: TEXT                 │
│    is_active: BOOLEAN     │        │    command: VARCHAR(100)          │
│    created_at: DATETIME   │        │    identifier: VARCHAR(200)      │
└──────────────────────────┘         │    is_confirmed: BOOLEAN         │
                                     │    corrected_text: TEXT           │
                                     │    created_at: DATETIME           │
                                     │    duration_seconds: INTEGER      │
                                     └──────────────────────────────────┘
```

### 3.2 Даталогическая модель

**Таблица `users`:**

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | INTEGER | PK, AUTO_INCREMENT | Идентификатор |
| username | VARCHAR(100) | UNIQUE, NOT NULL | Логин |
| hashed_password | VARCHAR(255) | NOT NULL | Хеш пароля (bcrypt) |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'operator' | Роль: admin / operator |
| is_active | BOOLEAN | DEFAULT TRUE | Активен / заблокирован |
| created_at | DATETIME | DEFAULT NOW | Дата создания |

**Таблица `voice_records`:**

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | INTEGER | PK, AUTO_INCREMENT | Идентификатор записи |
| user_id | INTEGER | FK → users.id, NOT NULL | Кто записал |
| audio_path | VARCHAR(500) | NOT NULL | Путь к аудиофайлу |
| raw_text | TEXT | NOT NULL | Результат распознавания |
| command | VARCHAR(100) | NULLABLE | Извлечённая команда |
| identifier | VARCHAR(200) | NULLABLE | Извлечённый идентификатор |
| is_confirmed | BOOLEAN | DEFAULT FALSE | Подтверждение оператора |
| corrected_text | TEXT | NULLABLE | Скорректированный текст |
| created_at | DATETIME | DEFAULT NOW | Время записи |
| duration_seconds | INTEGER | NULLABLE | Длительность аудио (сек.) |

**Связи:** User (1) → (*) VoiceRecord через `user_id`

## 5. Инструкция по развёртыванию

### Вариант 1: Docker (рекомендуется)

```bash
git clone <repo-url>
cd voice-command-system
docker-compose up --build
```

Фронтенд: http://localhost:3000
Бэкенд API: http://localhost:8000/docs

### Вариант 2: Локальный запуск

**Предварительные требования:**
- Python 3.11+
- Node.js 18+
- ffmpeg

**Backend:**
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Скачать модель VOSK
wget https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip
unzip vosk-model-small-ru-0.22.zip
mv vosk-model-small-ru-0.22 model

uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

### Учётные данные по умолчанию:
- **Логин:** admin
- **Пароль:** admin
- **Роль:** Администратор

## 6. API-эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| POST | /api/auth/login | Авторизация (OAuth2) |
| POST | /api/auth/register | Регистрация |
| GET | /api/auth/me | Текущий пользователь |
| POST | /api/records/upload | Загрузка аудиозаписи + распознавание |
| GET | /api/records | Список записей (с фильтрами) |
| GET | /api/records/{id}/audio | Скачать аудиофайл |
| PUT | /api/records/{id}/correct | Корректировка результата |
| PUT | /api/records/{id}/confirm | Подтверждение результата |
| GET | /api/users | Список пользователей (admin) |
| POST | /api/users | Создать пользователя (admin) |
| PUT | /api/users/{id} | Обновить пользователя (admin) |

## 7. Распознаваемые команды

- **Зарегистрировать** — регистрация объекта
- **Начать обработку** — начало обработки
- **Отменить обработку** — отмена обработки
- **Отменить регистрацию** — отмена регистрации
- **Завершить обработку** — завершение обработки

Примеры: «Зарегистрировать трубу номер Р45345ИВ», «Отменить обработку плавки 21957898»
