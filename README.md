# Система распознавания голосовых команд

## 1. Схема решения

Система построена по клиент-серверной архитектуре с разделением на слои:

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Клиент (Браузер)                            │
│  ┌───────────────┐  ┌──────────────────┐  ┌────────────────────┐   │
│  │  LoginPage    │  │   MainPage       │  │   HistoryPage      │   │
│  │  (авторизация)│  │  (запись+распозн)│  │  (фильтры+карточки)│   │
│  └───────────────┘  └──────────────────┘  └────────────────────┘   │
│                MediaRecorder API (WebM/Opus)                        │
└───────────────────────────┬─────────────────────────────────────────┘
                            │ REST API (JSON + multipart/form-data)
                            │ Authorization: Bearer <JWT>
┌───────────────────────────▼─────────────────────────────────────────┐
│                      Сервер (FastAPI)                               │
│  ┌─────────────┐  ┌──────────────────┐  ┌───────────────────────┐  │
│  │ auth_router  │  │ records_router   │  │   users_router        │  │
│  │ (JWT/bcrypt) │  │ (upload/CRUD)    │  │   (admin CRUD)        │  │
│  └─────────────┘  └────────┬─────────┘  └───────────────────────┘  │
│                             │                                       │
│              ┌──────────────▼───────────────┐                      │
│              │     Модуль распознавания      │                      │
│              │  ┌────────┐  ┌────────────┐  │                      │
│              │  │ ffmpeg  │  │   VOSK     │  │                      │
│              │  │ WebM→WAV│  │  ASR engine│  │                      │
│              │  └────────┘  └────────────┘  │                      │
│              │  ┌────────────────────────┐   │                      │
│              │  │  Парсер команд (regex) │   │                      │
│              │  └────────────────────────┘   │                      │
│              └──────────────────────────────┘                      │
│                             │                                       │
│              ┌──────────────▼───────────────┐                      │
│              │     Хранилище данных          │                      │
│              │  ┌────────┐  ┌────────────┐  │                      │
│              │  │ SQLite  │  │ Файловая   │  │                      │
│              │  │(метаданн)│ │ система    │  │                      │
│              │  │         │  │ (аудио)    │  │                      │
│              │  └────────┘  └────────────┘  │                      │
│              └──────────────────────────────┘                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Поток обработки голосовой команды:
1. Оператор авторизуется → получает JWT-токен
2. Нажимает «Начать запись» → браузер через MediaRecorder API записывает аудио (WebM/Opus)
3. Нажимает «Остановить» → аудиофайл отправляется на сервер через `POST /api/records/upload`
4. Сервер: ffmpeg конвертирует WebM → WAV (16kHz, mono)
5. Сервер: VOSK распознаёт WAV → текст на русском языке
6. Сервер: парсер извлекает команду и идентификатор из текста
7. Аудиофайл сохраняется на диск, метаданные — в SQLite
8. Результат возвращается клиенту → отображается с аудиоплеером
9. Оператор может прослушать, скорректировать или подтвердить результат

## 3. Структура базы данных

### 3.1 Инфологическая модель (ER-диаграмма)

```
┌──────────────────────────────┐             ┌──────────────────────────────────────┐
│           User               │             │           VoiceRecord                │
├──────────────────────────────┤             ├──────────────────────────────────────┤
│ PK  id          INTEGER      │  1       *  │ PK  id               INTEGER        │
│     username    VARCHAR(100)  │◄───────────┤ FK  user_id           INTEGER        │
│     hashed_pwd  VARCHAR(255)  │             │     audio_path        VARCHAR(500)   │
│     role        VARCHAR(20)   │             │     raw_text          TEXT            │
│     is_active   BOOLEAN       │             │     command           VARCHAR(100)    │
│     created_at  DATETIME      │             │     identifier        VARCHAR(200)    │
└──────────────────────────────┘             │     is_confirmed      BOOLEAN         │
                                              │     corrected_text    TEXT            │
                                              │     created_at        DATETIME        │
                                              │     duration_seconds  INTEGER         │
                                              └──────────────────────────────────────┘
```

**Связь:** User (1) ──── (*) VoiceRecord через внешний ключ `user_id`.
Один пользователь может иметь множество голосовых записей. Каждая запись принадлежит ровно одному пользователю.

### 3.2 Даталогическая модель

**Таблица `users`** — учётные записи операторов и администраторов:

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | INTEGER | PK, AUTOINCREMENT | Уникальный идентификатор |
| username | VARCHAR(100) | UNIQUE, NOT NULL, INDEX | Логин пользователя |
| hashed_password | VARCHAR(255) | NOT NULL | Хеш пароля (bcrypt, cost=12) |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'operator' | Роль: `admin` или `operator` |
| is_active | BOOLEAN | DEFAULT TRUE | Флаг активности (блокировка) |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Дата регистрации |

**Таблица `voice_records`** — результаты распознавания голосовых команд:

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| id | INTEGER | PK, AUTOINCREMENT | Уникальный идентификатор записи |
| user_id | INTEGER | FK → users.id, NOT NULL, INDEX | Кто выполнил запись |
| audio_path | VARCHAR(500) | NOT NULL | Имя файла в каталоге `uploads/` |
| raw_text | TEXT | NOT NULL | Результат распознавания VOSK |
| command | VARCHAR(100) | NULLABLE | Извлечённая команда (одна из 5) |
| identifier | VARCHAR(200) | NULLABLE | Извлечённый идентификатор |
| is_confirmed | BOOLEAN | DEFAULT FALSE | Подтверждение оператором |
| corrected_text | TEXT | NULLABLE | Скорректированный оператором текст |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Время выполнения записи |
| duration_seconds | INTEGER | NULLABLE | Длительность аудиозаписи (секунды) |

**Индексы:** `users.username` (UNIQUE), `voice_records.user_id` (FK INDEX), `voice_records.created_at` (для фильтрации по дате).

### 4.3 Описание сценариев

**Сценарий 1: Оператор распознаёт команду**
1. Оператор вводит логин/пароль → система выдаёт JWT-токен
2. На главной странице нажимает «Начать запись» → браузер активирует микрофон
3. Оператор произносит команду, например: *«Зарегистрировать трубу номер Р45345ИВ»*
4. Нажимает «Остановить» → аудио отправляется на сервер
5. Система отображает: распознанный текст, команду (`зарегистрировать`), идентификатор (`Р45345ИВ`)
6. Запись автоматически сохраняется в истории

**Сценарий 2: Корректировка неоднозначного результата**
1. Оператор видит результат распознавания → проверяет правильность
2. Если неверно: нажимает «Скорректировать» → прослушивает аудио через встроенный плеер
3. Редактирует текст, выбирает правильную команду из списка, вводит верный идентификатор
4. Нажимает «Сохранить» → затем «Подтвердить результат»
5. Подтверждение сохраняется в БД (поле `is_confirmed = true`)

**Сценарий 3: Поиск и верификация**
1. Оператор/админ переходит на страницу «История»
2. Использует фильтры: команда (dropdown), идентификатор, диапазон дат, имя оператора
3. Раскрывает карточку записи → видит все детали: текст, команду, ID, время, оператора, статус
4. Может прослушать аудио и при необходимости скорректировать

**Сценарий 4: Администрирование пользователей**
1. Админ переходит на страницу «Пользователи»
2. Создаёт нового оператора: вводит логин, пароль, выбирает роль
3. Может изменить роль существующего пользователя (dropdown «Оператор»/«Администратор»)
4. Может заблокировать/разблокировать учётную запись → изменения применяются мгновенно
5. Заблокированный пользователь не может войти в систему

## 5. Алгоритм распознавания и парсинга команд

### 5.1 Конвейер обработки

```
Аудио (WebM) ──► ffmpeg (-ar 16000 -ac 1) ──► WAV 16kHz mono ──► VOSK KaldiRecognizer
                                                                        │
                                                              текст на русском
                                                                        │
                                                                        ▼
                                                               ┌────────────────┐
                                                               │ extract_command │
                                                               │  _and_id()     │
                                                               └───────┬────────┘
                                                                       │
                                                          ┌────────────┼────────────┐
                                                          │            │            │
                                                     1. Поиск    2. Очистка   3. Конвертация
                                                     команды     от филлеров  слов → символы
                                                          │            │            │
                                                          ▼            ▼            ▼
                                                     "зарегистр." "эр 45 345  "Р45345ИВ"
                                                                   и вэ"
```

### 5.2 Обработка цифр

VOSK может выдавать числа в разных форматах:
- Поцифренно: *«два один девять пять семь восемь девять восемь»* → `21957898`
- Составными: *«двадцать один»* → `21`, *«пятьсот сорок пять»* → `545`
- Смешанно: *«сорок пять триста сорок пять»* → `45345`

Парсер поддерживает: единицы (0–9), десятки (10–90), сотни (100–900) и их комбинации.

### 5.3 Обработка букв

VOSK произносит буквы по их названиям: *«эр»* → `Р`, *«вэ»* → `В`, *«эс»* → `С`.
Парсер содержит словарь из 33 русских букв и их фонетических названий.

### 5.4 Распознаваемые команды

| Команда | Варианты (aliases) |
|---------|-------------------|
| Зарегистрировать | зарегистрируй, зарегистрируйте, регистрация |
| Начать обработку | начни обработку, начните обработку |
| Отменить обработку | отмени обработку, отмените обработку |
| Отменить регистрацию | отмени регистрацию, отмените регистрацию |
| Завершить обработку | заверши обработку, завершите обработку |

## 6. Инструкция по развёртыванию

### Вариант 1: Docker (рекомендуется)

```bash

cd voice-command-system
docker-compose up --build
```

- API (Swagger): http://localhost:8000/docs

**Backend:**
```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Скачать модель VOSK (русский)
```
